NEXT STEPS FOR DEPLOYMENT
================================================================================

1. PUSH TO GITHUB
   git add .
   git commit -m "Fix: Render deployment configuration and database persistence"
   git push origin main

2. PREPARE CREDENTIALS (5 minutes)
   - Gmail App Password: https://myaccount.google.com/apppasswords
   - reCAPTCHA Keys: https://www.google.com/recaptcha/admin
   - Generate Secret Key: python -c "import os; print(os.urandom(32).hex())"

3. CREATE RENDER SERVICE
   - Go to https://dashboard.render.com
   - Click "New +" → "Web Service"
   - Connect your GitHub repository
   - Follow QUICK_START_RENDER.md for configuration

4. CONFIGURE ENVIRONMENT VARIABLES
   Set these in Render dashboard:
   - DEV_MODE=false
   - SECRET_KEY=<generated-key>
   - EMAIL_USER=<your-gmail@gmail.com>
   - EMAIL_APP_PASSWORD=<app-password>
   - EMAIL_FROM=Your Name <your-email@gmail.com>
   - RECAPTCHA_SITE_KEY=<your-site-key>
   - RECAPTCHA_SECRET_KEY=<your-secret-key>

5. CREATE PERSISTENT DISK
   - Name: sqlite-db
   - Mount Path: /var/data
   - Size: 1 GB

6. DEPLOY
   - Click "Deploy" in Render dashboard
   - Monitor logs for any errors
   - Test at https://<service-name>.onrender.com

TROUBLESHOOTING QUICK REFERENCE
================================================================================

Database Not Persisting?
  → Ensure persistent disk is created at /var/data
  → Check Render logs for permission errors

Email Not Sending?
  → Verify Gmail App Password (not regular password)
  → Check SMTP settings in logs
  → Ensure email is enabled in Gmail security

reCAPTCHA Issues?
  → Add *.onrender.com to reCAPTCHA domains
  → Verify keys are correct
  → Test keys show "testing purposes only" banner

Port Binding Errors?
  → Render automatically sets PORT environment variable
  → No manual configuration needed

For detailed troubleshooting, see: RENDER_DEPLOYMENT.md
